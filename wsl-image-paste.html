Image Paste for Claude Code on WSL

Paste screenshots directly into Claude Code from your Windows clipboard.

The problem

Windows captures screenshots as BMP, but Claude Code expects PNG. On WSL, pasting an image with Ctrl+V doesn't work because Windows Terminal intercepts the keystroke. This guide installs a lightweight wrapper that bridges both gaps.

Drag and drop already works: You can drag an image file from Windows Explorer directly into the Claude Code terminal window. This guide is specifically for clipboard paste (screenshots, snipping tool, copied images).
File paths work too: You can reference any image by path in your message and Claude will read it. Use /mnt/c/ to reach Windows files, e.g. /mnt/c/Users/you/Pictures/screenshot.png.

Prerequisites (run this yourself)

The install script needs sudo to install dependencies and write to /usr/local/bin/.

wget -qO /tmp/install-wl-paste-wrapper.sh https://alexqhj.github.io/static/assets/install-wl-paste-wrapper.sh && bash /tmp/install-wl-paste-wrapper.sh

This installs imagemagick and wl-clipboard, then places a thin wrapper at /usr/local/bin/wl-paste that converts BMP clipboard images to PNG on the fly.

Set up a keybinding for pasting images into Claude Code on WSL.

Windows Terminal intercepts Ctrl+V, so we need a different key for image paste. The default is Alt+V, but you can change it below.

1. Create (or update) ~/.claude/keybindings.json with this content:
   {
     "bindings": [
       {
         "context": "Chat",
         "bindings": {
           "alt+v": "chat:imagePaste"
         }
       }
     ]
   }

   If the file already exists, merge the new binding into the existing bindings — don't overwrite other keybindings.

2. IMPORTANT: Tell me to fully quit and restart Claude Code for the keybinding to take effect.
   After restarting, resume with "claude --continue" so we can verify together.

3. After restart, tell me to take a screenshot with Win+Shift+S and press Alt+V to test the image paste.

Usage

1

Take a screenshot

Press Win+Shift+S to open the Snipping Tool, or copy any image to clipboard.

2

Paste into Claude Code

Press Alt+V (or your custom keybinding) in the Claude Code prompt. The image will appear inline.

Custom keybinding

If you'd prefer a different key combination, edit ~/.claude/keybindings.json and change "alt+v" to your preferred binding. Examples:

"alt+v": "chat:imagePaste"           // default
"ctrl+shift+v": "chat:imagePaste"    // Ctrl+Shift+V
"alt+i": "chat:imagePaste"           // Alt+I for "image"
"alt+p": "chat:imagePaste"           // Alt+P for "paste"

Restart Claude Code after changing keybindings.

How it works

The wrapper: A thin script at /usr/local/bin/wl-paste that intercepts calls from Claude Code. When the clipboard contains a BMP image (Windows default), it converts it to PNG via ImageMagick. All other clipboard operations pass through to the real /usr/bin/wl-paste unchanged.
The keybinding: Windows Terminal captures Ctrl+V before it reaches Claude Code, so we bind chat:imagePaste to Alt+V instead. This tells Claude Code to read the clipboard image.

Uninstall

To remove the image paste setup:

sudo rm /usr/local/bin/wl-paste

To also remove the keybinding, edit ~/.claude/keybindings.json and delete the "alt+v": "chat:imagePaste" entry.
