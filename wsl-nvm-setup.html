Set up nvm with Node.js on my WSL environment. Do the following steps:

1. Install nvm:
   - Run: curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash

2. Install the latest LTS version of Node.js and set it as default:
   - Source nvm first: export NVM_DIR="$HOME/.nvm" && [ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
   - Run: nvm install --lts
   - Run: nvm alias default node

3. Add lazy-loaded nvm to ~/.zshrc (or ~/.bashrc if zsh is not installed).
   Remove any existing nvm initialization lines that the nvm installer added (the "export NVM_DIR" block).
   Replace them with this lazy-loading setup that saves ~300ms on shell startup:

   export NVM_DIR="$HOME/.nvm"

   # Lazy load NVM - don't source it at startup (saves ~300ms)
   nvm() {
     unset -f nvm node npm npx
     [ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
     [ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"
     nvm "$@"
   }

   # Lazy load node/npm/npx too
   for cmd in node npm npx; do
     eval "$cmd() { unset -f nvm node npm npx; [ -s \"\$NVM_DIR/nvm.sh\" ] && \. \"\$NVM_DIR/nvm.sh\"; $cmd \"\$@\"; }"
   done

4. Add auto-switching to ~/.zshrc (or ~/.bashrc) AFTER the lazy loading block from step 3.
   This automatically switches Node versions when you cd into a project that has a .node-version or .nvmrc file:

   # Auto-switch Node version when entering a project directory
   autoload -U add-zsh-hook
   load-nvmrc() {
     local dir="$PWD"
     local nvmrc_path=""

     # Search up directory tree for .nvmrc or .node-version
     while [[ "$dir" != "/" ]]; do
       if [[ -f "$dir/.nvmrc" ]]; then
         nvmrc_path="$dir/.nvmrc"
         break
       elif [[ -f "$dir/.node-version" ]]; then
         nvmrc_path="$dir/.node-version"
         break
       fi
       dir="${dir:h}"
     done

     if [[ -n "$nvmrc_path" ]]; then
       local requested=$(< "$nvmrc_path")
       requested="${requested//[$'\t\r\n ']/}"

       if type nvm &>/dev/null && [[ "$(type -w nvm)" != "nvm: function" || "$(nvm current 2>/dev/null)" != "" ]]; then
         local current=$(nvm current 2>/dev/null)
         if [[ -n "$current" && "$current" != "$requested" && "$current" != "v$requested" ]]; then
           echo "Switching to Node $requested..."
           nvm use "$requested" 2>/dev/null || nvm install "$requested"
         fi
       fi
     elif type nvm &>/dev/null && [[ "$(type -w nvm)" != "nvm: function" ]]; then
       local current=$(nvm current 2>/dev/null)
       if [[ -n "$current" && "$current" != "default" && "$current" != "system" ]]; then
         echo "Reverting to default Node..."
         nvm use default 2>/dev/null
       fi
     fi
   }

   add-zsh-hook chpwd load-nvmrc
   load-nvmrc

   NOTE: If the user has bash instead of zsh, replace "autoload -U add-zsh-hook", "add-zsh-hook chpwd load-nvmrc", and "${dir:h}" with bash equivalents (use PROMPT_COMMAND and dir="$(dirname "$dir")").

5. Verify the setup:
   - Run: zsh -c 'source ~/.zshrc && node --version && echo "nvm dir: $NVM_DIR"'
   - Tell me to open a new terminal and cd into a project with a .node-version file to test auto-switching.

What this sets up

Lazy loading: nvm normally adds ~300ms to every shell startup. The lazy-loading wrapper defers sourcing nvm until you actually run nvm, node, npm, or npx for the first time in a session.
Auto-switching: When you cd into a project that has a .node-version or .nvmrc file, nvm automatically switches to the correct version. It walks up the directory tree, so it works from subdirectories too. If the version isn't installed, it installs it automatically.
Reverting: When you leave a project directory (cd somewhere without a .node-version file), it reverts to your default Node version.

Adding a version to your project

To pin a Node version for your project, create a .node-version file in the project root:

echo "22.21.1" > .node-version

This ensures everyone on the team uses the same Node version. Commit this file to your repo.
