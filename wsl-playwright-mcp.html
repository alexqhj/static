Playwright MCP for WSL

Let Claude Code control your Windows Chrome browser from WSL using the Playwright MCP extension.

Prerequisites (do these yourself)

These steps require manual action in your Windows Chrome browser.

1

Install the Chrome extension

Install Playwright MCP Bridge from the Chrome Web Store.

2

Get your extension token

Click the Playwright MCP Bridge extension icon in your Chrome toolbar, or navigate to chrome-extension://mmlmfjhmonkocbjadbfplnigmagldckm/status.html. Copy the PLAYWRIGHT_MCP_EXTENSION_TOKEN value — you'll paste this into the Claude Code prompt below.

3

Make sure Chrome is running

The extension mode connects to your already-running Chrome — it doesn't launch a new instance.

Claude Code prompt

Copy this prompt and paste it into Claude Code. Replace YOUR_TOKEN_HERE with the token from step 2 above.

Set up Playwright MCP to control my Windows Chrome from WSL using the browser extension. The Playwright MCP Bridge Chrome extension is already installed and Chrome is running.

My extension token is: YOUR_TOKEN_HERE

Do the following steps:

1. Find my global MCP config file:
   - It's at ~/.mcp.json (this is where Claude Code reads MCP server config)

2. Add or update the playwright MCP server entry in the mcpServers object:
   {
     "playwright": {
       "type": "stdio",
       "command": "npx",
       "args": [
         "-y", "@playwright/mcp@next",
         "--extension",
         "--executable-path", "/mnt/c/Program Files/Google/Chrome/Application/chrome.exe"
       ],
       "env": {
         "PLAYWRIGHT_MCP_EXTENSION_TOKEN": "YOUR_TOKEN_HERE"
       }
     }
   }
   - Replace YOUR_TOKEN_HERE with the token I provided above
   - The "command" value for npx should use the full path from: which npx
   - Add the npx directory to the PATH env var as well
   - Merge this into the existing config — don't overwrite other MCP servers
   - If there's already a playwright entry, replace it entirely

3. Tell me to fully quit and restart Claude Code.
   IMPORTANT: A /mcp reconnect is NOT enough — you must exit Claude Code completely and relaunch it for MCP config changes to take effect.
   After restarting, resume this conversation with "claude --continue" so we can verify together.

4. After restart, verify by asking me to test the browser_navigate tool on any URL.

How it works

Extension mode: The MCP Bridge extension lets Claude connect to your existing Windows Chrome. This means Claude can interact with pages where you're already logged in — your cookies, sessions, and browser state are all available.
WSL → Windows: The --executable-path flag tells Playwright where to find Chrome on Windows via /mnt/c/. No browser needs to be installed inside WSL.
Token: The PLAYWRIGHT_MCP_EXTENSION_TOKEN auto-approves connections so you don't have to click "Allow" in Chrome every time Claude uses the browser. Click the extension icon or visit chrome-extension://mmlmfjhmonkocbjadbfplnigmagldckm/status.html to find it.

Troubleshooting

If Playwright can't connect:

  1. Make sure Chrome is open and the extension is enabled
  2. Check the token hasn't changed — it regenerates if you reinstall the extension
  3. Fully quit and restart Claude Code after any config change — /mcp reconnect is not enough
  4. Verify @playwright/mcp@next is being used, not @latest — the WSL executable-path fix may not be in the stable release yet
